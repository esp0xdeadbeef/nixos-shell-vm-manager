# FS-160-HDS-010 Optional Pin-refresh Construction Environment

The host generation retains each enabled VM's declaratively approved flake
reference and build target alongside its host-pinned image. Pin-refresh runtime
resolution archives that reference immutably before creating a writable refresh
checkout. Prefer a revision-pinned remote reference so the approved source is
reproducible without retaining the complete consumer repository in the host
generation.

Pin refresh uses an isolated writable workspace in replaceable host storage,
the Nix daemon and optional network access for dependency resolution, the
per-VM construction lock, and the shared host build-capacity pool. Its immutable
archive, refreshed lock identity, and completed output cross the candidate
admission boundary only after successful construction.

The host-pinned image and admitted registry remain locally available throughout
refresh. Network, resolution, capture, or build failure cannot remove that
offline fallback. Rollback and recovery use retained local artifacts and do not
enter the refresh environment.
