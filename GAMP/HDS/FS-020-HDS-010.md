# FS-020-HDS-010 Offline Runtime Closure

The activated host retains the selected image, its runner closure, manager
programs, QEMU dependencies, storage tools, and the configured health command.
The runtime service has no source or build input and shall fail visibly when a
registered local artifact is absent. A self-hosted connectivity VM starts from
this closure without connectivity supplied by the VM itself. Its automatic
activation policy orders only after declared host-local interfaces and bridges;
network-dependent maintenance is not part of the bootstrap dependency chain.
