# FS-160-HDS-010-SDS-010-SMS-010 Pin-refresh Construction Module

`modules/default.nix` emits `REFRESH_PINS`, `PIN_REFRESH_FLAKE`, and
`PIN_REFRESH_FLAKE_ATTRIBUTE` from evaluated, assertion-checked instance
options. `scripts/nixos-shell-vm-manager.sh` implements an eligible-event gate
and a best-effort construction subprocess.

The subprocess serializes construction, copies the immutable source to a
validated temporary workspace, makes only that copy writable, updates its lock,
archives and builds without additional lock mutation, computes a lock hash, and
admits one validated output as `pin-refresh`. Cleanup closes file descriptors
and removes the workspace on both success and failure.

Failure Conditions:

- A disabled or excluded event invokes Nix.
- The immutable configured source or host lock is modified.
- A failed update, archive, build, validation, or admission changes a slot.
- Failure prevents normal host-pinned selection.
- The output omits pin-refresh source or lock provenance.
- Construction bypasses the per-VM lock or shared token pool.
- Rollback or recovery enters the module.
