# FS-075-HDS-010-SDS-010-SMS-010 Authority Commands

Implements `authorize`, `prepare-start`, `stop`, and epoch comparison using
atomic files below the volatile control directory. Stop writes authority state
before signaling the supplied supervisor PID. Automatic starts pass the exact
authority epoch through `preauthorized`; `prepare-start` consumes it only when
the epoch remains current and no stop marker exists. Carrier-down records its
typed marker and a matching short-lived automatic-stop token before sending the
supervisor's dedicated signal. The resulting systemd `ExecStop` consumes that
token without changing the stop type; an `ExecStop` without a current token
records explicit operator authority.
