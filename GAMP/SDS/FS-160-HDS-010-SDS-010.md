# FS-160-HDS-010-SDS-010 Serialized Pin-refresh Pipeline

The generated instance configuration provides an enabled bit, an immutable
approved flake source path, and a build attribute. The supervisor invokes the
pipeline for ordinary explicit starts, carrier-up starts, and guest-shutdown
restarts. A boot start uses the same ordinary service-start event.
Force-candidate rollout, local update, rollback, and recovery do not invoke it.

The pipeline acquires the per-VM construction lock followed by one shared build
token. It copies the approved source to a new writable replaceable workspace,
updates that copy's dependency lock according to its declared inputs, archives
the refreshed source without further lock mutation, hashes the effective lock,
and builds exactly one output from the configured attribute without lock writes.
It validates and admits that output as source kind `pin-refresh` with immutable
source and lock identities.

Refresh execution is a best-effort subprocess. Any resolution, capture,
construction, validation, or admission failure closes its locks, removes the
workspace, leaves every registry slot unchanged, and returns control to normal
host-pinned image selection. Successful admission precedes normal selection and
health-gated activation. Stop authority is checked immediately before launch.
