# FS-070-HDS-010-SDS-010 Foreground Lifecycle Supervisor

The supervisor selects candidate/current according to the event and per-VM
policy, rechecks stop authority under lock, launches the exact image runner in
the foreground service, and owns guest-shutdown recovery. `SIGUSR1` requests an
explicit candidate rollout. `SIGTERM` records stopping and terminates the child
without recovery. Guest recovery waits a random inclusive configured delay.
Before an eligible explicit or guest-shutdown start, the supervisor invokes the
FS-160 pin-refresh pipeline only when `refreshPins` is enabled. Force-candidate,
rollback, and recovery transitions bypass that pipeline.
