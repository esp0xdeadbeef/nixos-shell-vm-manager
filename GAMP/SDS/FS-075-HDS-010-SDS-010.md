# FS-075-HDS-010-SDS-010 Session-authority Protocol

An authority operation increments a per-session epoch and removes the stop
marker. An operator stop increments it and atomically records `explicit-stop`
before the runner is terminated. The carrier adapter records `carrier-down`
through the same protocol; carrier-up preauthorization may clear only that
automatic reason and returns a policy refusal for `explicit-stop`. Repeated
handling of the same stop is idempotent. ExecStartPre consumes automatic
preauthorization only when its epoch is still current and no later stop marker
exists. Carrier-down uses a dedicated supervisor signal rather than the
operator `ExecStop` path. A stop observed during candidate health leaves that
candidate pending, does not start recovery, and records the registry phase as
`idle`. `vm-update` may roll out only if its recorded epoch is still current and
no marker exists. Supervisor start performs the same marker check immediately
before each child launch.
