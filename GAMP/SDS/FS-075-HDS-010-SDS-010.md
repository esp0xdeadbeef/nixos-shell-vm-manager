# FS-075-HDS-010-SDS-010 Session-authority Protocol

An authority operation increments a per-session epoch and removes the stop
marker. A stop operation increments it and atomically creates the marker before
the runner is terminated. Repeated handling of the same stop is idempotent. A
stop observed during candidate health leaves that candidate pending, does not
start recovery, and records the registry phase as `idle`. `vm-update` may roll
out only if its recorded epoch is still current and no marker exists.
Supervisor start performs the same marker check immediately before each child
launch.
